Git info

Git is a distributed version control system that tracks changes in any set of computer files, usually used for coordinating work among programmers collaboratively developing source code during software development. Its goals include speed, data integrity, and support for distributed, non-linear workflows (thousands of parallel branches running on different systems).

Git commands

  1. Git clone - This command is for downloading existing source code from a remote repository.
  2. Git branch - This command shows branches in the project. They are highly important in the git world. By using branches, several developers are able to work in parallel on the same project simultaneously: git branch (branch-name)
  3. Git checkout - This command is being used to swap between the branches.
  4. Git status - The Git status command gives us all the necessary information about the current branch.
  5. Git add - This command includes the changes of a file(s) into our next commit. We can use git add (file) for a specific file or git add . to add all changes.
  6. Git commit - This command is used to save our changes. Git commit is like setting a checkpoint in the development process which you can go back to later if needed. + We need to leave a comment: git commit -m "commit message"
  7. Git push - This command is used after commiting the changes to send them to the remote server. It upploads all the commits to the remote repository.
  8. Git pull - This command is used to get updates from the remote repository.
  9. Git revert - This command is used to undo the changes. To see our commit history, first we need to use git log -- oneline:
  10. Git merge - This command is used to merge the branch with a parent one. Git merge basically integrates your feature branch with all of its commits back to the dev (or master) branch.
